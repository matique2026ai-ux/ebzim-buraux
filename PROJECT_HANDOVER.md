# وثيقة تسليم المشروع - جمعية إبزيم
**التاريخ: 15 أبريل 2026**

هذه الوثيقة تلخص الحالة الراهنة للمشروع وما تم إنجازه من تنظيم وإعادة هيكلة، لتسهيل متابعة العمل من أي مكان آخر أو بواسطة وكيل ذكاء اصطناعي آخر.

## 1. الحالة الراهنة للمشروع
المشروع الآن منجم ومنظم بشكل احترافي، وتم فصل المكونات بشكل هرمي واضح. المشروع مخصص حالياً لمنصات **الهواتف (Android/iOS)** و **الويب (Web)**.

### الهيكل التنظيمي الجديد:
- **`lib/`**: الكود البرمجي الخاص بتطبيق Flutter (الميزات، الواجهات، والمنطق).
- **`backend/`**: سيرفر NestJS (نظام إدارة المحتوى - CMS).
  - `backend/scripts/`: تحتوي على سكربتات الصيانة والبيانات الأولية (Seeds).
  - `backend/.env.example`: نموذج لإعدادات البيئة (يجب تعبئته للتشغيل).
- **`assets/`**: الموارد المنظمة.
  - `assets/data/seeds/`: ملفات JSON الخاصة بالبيانات الأولية.
  - `assets/css/`: ملفات التنسيق الإضافية (مثل `fonts.css`).
  - `assets/fonts/`, `assets/images/`, `assets/icons/`.
- **`scripts/`**: سكربتات تشغيل Flutter Web (`serve_static.js`, `serve_web.js`).
- **`docs/`**: التوثيق.
  - `docs/design/`: ملفات التصميم والمراجع.

## 2. كيفية التشغيل
للمتابعة من حيث توقفنا، اتبع الخطوات التالية:

### تشغيل الباك اند (Backend):
1. انتقل إلى مجلد الباك اند: `cd backend`
2. تأكد من وجود ملف `.env` (استخدم `.env.example` كنموذج).
3. قم بتشغيل السيرفر: `npm run start:dev`
4. الباك اند متاح على: `http://localhost:3000`

### تشغيل التطبيق (Flutter Web):
1. من المجلد الرئيسي للمشروع:
2. قم بتشغيل التطبيق: `flutter run -d web-server --web-port 5050`
3. التطبيق متاح على: `http://localhost:5050`
